## Do / Don’t visual anchors

Use these filenames during design review so feedback references the same snapshot.

### Do

- `ProMarked.png`: keep dense but legible panel hierarchy, with clear information zoning.
- `automix.png`: prioritize transition status and automation affordances near transport controls.
- `library.png`: preserve browser scanability with stable column rhythm and obvious active-row highlight.

### Don’t

- `ProMarked.png`: do not mirror every legacy ornament, bevel, or decorative chrome 1:1.
- `automix.png`: do not hide transport or automix state behind secondary menus.
- `library.png`: do not overload the browser with low-priority metadata at the expense of quick song selection.

## Non-goals

- Reproducing legacy skins pixel-for-pixel.
- Importing dated visual debt (excessive gradients, noisy skeuomorphic textures, inconsistent iconography).
- Expanding feature scope based solely on what appears in third-party references.
- Treating downloaded web artifact files under `rekordbox 7 Overview _ rekordbox - DJ software_files/` as direct UI implementation templates.
